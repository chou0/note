* 二维数组底层实际上是一维的，因为内存就是一维的

## 2x3 还是 3x2
```
int a[2][3] = 
{
    {1, 2},
    {3, 4},
    {5, 6}
};
int b[2][3] =
{
    {1, 2, 3},
    {4, 5, 6}
};
int c[3][2] =
{
    {1, 2, 3},
    {4, 5, 6}
};
int c[3][2] =
{
    {1, 2},
    {3, 4},
    {5, 6}
};
```
* 上面 4种写法都 ok吗？
* 4种写法，编译器都不会报错，原因是 底层是一维表示的， 2x3 和 3x2 占用内存都是一样的；但是访问却是不同的，很容易出问题
例如：
```
a[2][3]

```
* 按照我初始逻辑，认为是 3个2 因此表示成 a 写法，访问 a[1][0] 应该是 3，但是实际测试却不对
* 测试发现， 编译器逻辑认为 a[2][3] 写法是 2个3， 应该写成b方式，并且按照 2个3 的方式的读取才是ok的
